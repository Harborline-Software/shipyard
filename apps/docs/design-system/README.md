# Sunfish Shared Design System

The Sunfish Shared Design System ([ADR 0077](../../docs/adrs/0077-shared-design-system.md))
is a three-package substrate for accessible, role-aware, conformance-tracked UI:

| Package | What it provides | Phase |
|---|---|---|
| `Sunfish.Foundation.Ship.Common` | `ShipRole` taxonomy · `IDeckRegistry` · `IPermissionResolver` · `IShipRoleRegistry` | W#46 Phase 1 |
| `Sunfish.UICore` | `IConformanceRegistry` · `ILiveAnnouncer` · `IFocusTrap` · `IFirstAidContract` contracts | W#46 Phase 3 |
| `Sunfish.UIAdapters.Blazor` / `Sunfish.UIAdapters.React` | Concrete adapter implementations | W#46 Phase 4 |

## Registration

### Blazor (Anchor / kitchen-sink)

```csharp
// Program.cs
builder.Services.AddSunfishSharedDesignSystem();  // Phase 1 + Phase 3
builder.Services.AddSunfishA11y();                // Phase 4 — ILiveAnnouncer, IFocusTrap
```

### React (Bridge)

```tsx
// Root component
import { ConformanceRegistryProvider } from '@sunfish/ui-adapters-react/a11y';

<ConformanceRegistryProvider>
  <App />
</ConformanceRegistryProvider>
```

## Architecture

The Shared Design System follows the Sunfish framework-agnostic principle:

1. **Define the contract** in `foundation-ship-common` (`ShipRole`, `IDeckRegistry`) and `ui-core` (`IConformanceRegistry`, `ILiveAnnouncer`)
2. **Implement in adapters** — Blazor and React adapters implement the contracts independently
3. **Compose in blocks** — W#35 cohort blocks (`blocks-quarterdeck`, `blocks-engine-room`, etc.) consume the contracts via DI

See the [architecture principles](../../../_shared/product/architecture-principles.md) for the layering invariants.
